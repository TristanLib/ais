## Publication Workflow

Use Python 3.11 and the project-local virtual environment:

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Run the strict full evidence pipeline:

```bash
PYTHON_BIN=.venv/bin/python bash scripts/run_final_experiment.sh
```

Run a smoke test only when checking infrastructure:

```bash
PYTHON_BIN=.venv/bin/python bash scripts/run_final_experiment.sh --max-train-samples 1000 --max-test-samples 100
```

Smoke-test numbers must not be used in the paper. Publication numbers are usable only when `outputs/final/run_manifest.json` has `is_debug_run=false` and `outputs/final/publication_readiness_report.json` has `status=pass`.

Run the higher-bar roadmap pipeline with data download enabled:

```bash
PYTHON_BIN=.venv/bin/python \
DOWNLOAD_DATES=true \
bash scripts/run_high_quality_pipeline.sh
```

For infrastructure checks only:

```bash
PYTHON_BIN=.venv/bin/python \
DOWNLOAD_DATES=true \
MAX_ROWS_PER_FILE=1000000 \
MAX_TRAIN_SAMPLES=512 \
MAX_TEST_SAMPLES=128 \
bash scripts/run_high_quality_pipeline.sh
```

High-quality numbers are citable only after all of these are true:

- `outputs/audit/multiday_data_manifest.json` covers at least four source dates.
- `outputs/final_multiday/run_manifest.json` has `is_debug_run=false`.
- `outputs/final_submission/readiness_report.json` has no `blocking_gaps`.
- `paper/submission_manuscript.md` and `paper/references.bib` exist.

## Metric and Data Rules

The final protocol is fixed at 30 one-minute input steps to 15 one-minute forecast steps. The processed arrays use WGS84 latitude/longitude; ADE/FDE are Haversine displacement errors in meters, and RMSE/MAE use local north/east component errors in meters.

The processed NPZ lacks MMSI, timestamp, vessel type, and region metadata. Do not claim temporal cross-validation, vessel-disjoint splits, regional generalization, or group-level robustness from this evidence pack.

The high-quality NPZ at `data/processed/multiday_high_quality_processed.npz`
does retain MMSI, timestamps, source date, region, speed, turn intensity, ship
type, interpolation ratio, and both temporal/vessel-disjoint split labels.
The latest high-quality build covers four NOAA source dates
(`2024-01-02`, `2024-01-09`, `2024-02-06`, `2024-03-05`) with a documented
stratified time-block protocol, computed raw checksums, and non-debug benchmark
metrics. Current high-quality headline result: `kalman_filter_cv` is best by ADE
on temporal holdout (1759.7 m) and vessel-disjoint holdout (3109.4 m). Do not
claim all-day traffic generalization, architecture superiority, or autonomous
collision avoidance from these artifacts without additional evidence.

Prefer the conservative final pipeline for publication claims. Legacy scripts such as `scripts/10_train_baselines.sh`, `scripts/20_train_transformer.sh`, and `scripts/30_eval_all.sh` can still be used for exploration, but their outputs are not manuscript evidence unless routed through the final artifact contract.

Use `rg` for searching. Keep generated evidence in `outputs/audit/` and `outputs/final/`. Do not manually edit generated tables, figures, or manuscript results; regenerate them from scripts.
